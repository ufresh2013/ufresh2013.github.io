Vue2, Vue3, React Class, React Hooks的区别

2024-08-25

都是MVC（Model-View-Controller）数据驱动的框架，Vue和React的区别是什么？分别好在哪里？

1. MVC

先回顾一下，啥是MVC?

MVC（Model-View-Controller） 是一种经典的软件架构模式，用于分离应用的数据、界面、逻辑

  • Model：管理数据
  • View：界面展示，从 Model 获取数据并呈现出来
  • Controller：接收用户输入，调用 Model 修改数据，更新 View 展示结果

我们先简单介绍一下Vue2, Vue3, React Class, React Hooks，最后再总结它们的区别。


2. Vue2

2016年Vue2发布。Vue2使用 Options API 配置化的方式组织代码，搭配各种API、指令，很容易满足了各种页面需求，上手容易，简单好用。


3. React(Class Component)

和 Vue2 同期的是 React 的 Class Component。

每个组件既是 <ReactComponent /> HTML的写法，又是一个类的实例，这就是Class Component。

类(class)让组件的数据和操作逻辑封装在一起，最后返回了JSX，又表达了这是一个UI组件。

hhh第一次看到这种代码组织方式就觉得很妙！


3. React16 (Function Component + Hooks)

React, Vue 的出世，搭配 Ant Design、Element UI 等组件库，大大解放了前端生产力。

正当大家用得很开心的时候，2019年 React 推出 Hooks 写法。


3.1 函数组件

React 希望，组件不要变成复杂的容器。

React 的函数组件只应该做一件事情：返回组件的 HTML 代码，而没有其他的功能。

但是，这种写法有很大的限制，纯函数无法包含状态，也不支持生命周期方法。

为了解决这个问题，React引入Hooks，希望不使用”类”，也能写出一个全功能的组件。


3.2 Hooks

Hooks 是钩子的意思，组件还是要写成纯函数，如果要其他功能，就要用钩子把它们”钩”进来。

常见的钩子有：

  • useState：为函数组件引入状态state
  • useEffect：引入具有副作用的操作，如获取数据、事件监听、改变DOM

你还可以创建自己的Hooks。

Hooks的写法将数据、逻辑从class中分离出来，开启了组件逻辑表达和逻辑复用的新范式。


4. Vue3 Compsition API

受到 React hooks 的启发，Vue3 提供了类似的 Composition API。通过 ref 和 reactive 管理数据。


5. Vue 和 React 的区别

好了，所以 Vue 和 React的区别是什么？

我认为有下面这几点：

1. 响应式的写法不同

  • Vue 用 state = newState 来通知变更。getter，setter 对用户不可见，更符合大家心目中的写法。

    但是无论是Vue2中 this.foo = a，还是 Vue3 中 foo.value = a 的写法，都又一些需要额外处理的地方，还不能完全达到 state = newState 的效果。如Vue2中往 data 新增属性，数组操作需要额外的API，Vue3中的 .value 写法不够直观，拆分了 ref 和 reactive 两种类型也很让人很困惑

  • React显式调用 setState(newState) 来通知变更，简单粗暴。

    当 state 是复杂对象时，复制一个全新的副本，修改它，是一件麻烦的事。不过 React 也不想管这个问题，它推荐你用 ImmerJS 来处理。


2. 实现响应式方式不同

  • 依赖收集是Vue的特点，Vue用 proxy , Object.defineProperty 实现依赖收集，依赖追踪，实现了性能更高的diff效果

  • React没有依赖收集，而是在 setState 后通过UI Tree一层一层往下更新渲染。

    当层级较高的 state 变化时，会引起大量组件的更新。为了解决这个问题，React 提出了 Fiber 来处理计算量大的情况下，没有交互响应间隙的问题。同时提供了 useMemo, useCallback 允许你缓存组件，减少不必要的渲染。


3. render 渲染写法不同

template 和 JSX 都是 render 的一种表现形式。

  • React 的 JSX 能完全融合到JS中，具有更高的灵活性，在复杂的组件中，更具有优势。
  • template 其实是往上封装了一层，相同的逻辑代码量更少，更简单直观、更好维护。template可以提供更好的性能优化，用起来也很贴合需求，不会觉得不够用。


4. API 提供的功能不同

  • React Hooks 寥寥几个API，Hooks + Effect 就能满足所有需求，灵活巧妙，需要你一点点去组装。
  • Vue 则是提供了大量的 API，如computed, watch, provide/inject, directive，keep-alive，生命周期。这些API是友好的，但是需要你一个个去熟悉，它们像一个个最佳实践，碰到什么场景就用什么。


5. 代码组织方式不同

  • React 全 JS 的写法，自成一体。函数组件 + hooks钩子 + UI Tree的设计理念，使得它写法很流畅。
  • Vue分离了<template>和<script>，逻辑和模板分离，加上大量封装程度更高的API。虽然Vue3加入了更灵活的Composition API，但你还是会感受到它本质还是有点opiton配置式的感觉



End.
