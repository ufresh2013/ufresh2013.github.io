React Router

2018-11-06

1. React-Router

React Router中有三类组件

  • router components
  • route matching components
  • and navigation components.


1.1 Routers

For web projects, react-router-dom provides <BrowserRouter> and <HashRouter> routers. Both of these will create a specialized history object for you.


1.2 Route Matching

There are two route matching components: <Route> and <Switch>. Route matching is done by comparing a ‘s path prop to the current location’s pathname. When a matches it will render its content. Then component is used to group s together.

A is useful. It will iterate over all of its children elements and only render the first one that matches the current location. This helps when multiple route’s paths match the same pathname, when animating transitions between routes, and in identifying when no routes match the current location(so that you can render a “404” component).

Route Rendering Props
You have three prop choices for how you render a component for a given : component, render and children.

component should be used when you have an existing component that you want to render. render, which takes an inline function, should only be used when you have to pass in-scope variables to the component you want to render.


1.3 Navigation

React Router provides a component to create links in your application. Wherever you render a , an anchor <a> will be rendered in your application’s HTML.

Any time that you want to force navigation, you can render a <Redirect>. When a renders, it will navigate using its to prop.
